# Task Book: Zen Campus Integrated Service System

## 1 Objectives

Deliver a production-like campus integrated service system with complete business workflows and no placeholder-only features.

## 2 Scope

1. Authentication and role control
   - Local account login/register
   - JWT authentication
   - Roles: student, teacher, admin

2. Business modules
   - Classroom reservation and approval
   - Lecture publish/signup/waitlist/check-in
   - Equipment borrowing and return workflow
   - Bus route/trip booking with waitlist promotion
   - Study-room seat reservation with real-time status

3. Operations and admin
   - Rule configuration in admin panel
   - Template-based site messages
   - Dashboard and report aggregation

## 3 Quality Requirements

- Concurrency-safe booking behavior
- Full workflow integrity (create, approve/reject, cancel, check-in/return)
- Successful backend/frontend build
- Documentation aligned with implemented code

## 4 Milestones

- M1: architecture and schema confirmation
- M2: core services delivered
- M3: admin/report completion
- M4: integration testing and bug fixing
- M5: thesis package and defense materials

## 5 Deliverables

- Source code
- SQL and configuration files
- Diagram set (01-09)
- Thesis/proposal/task-book documents
